‚ö° PoW Challenge Layer

Proof-of-Work protection against automated attacks, inspired by Anubis project

Idea: Make attackers spend computational resources. For legitimate users ‚Äî imperceptible ~100ms delay. For bots making 1000 req/sec ‚Äî prohibitive costs.
üßÆ

SHA256 Challenge

Client must find nonce so hash(challenge + nonce) starts with N zeros. Difficulty adapts to load.

üéØ

Rule-Based Triggers

PoW triggers based on rules: suspicious User-Agent, high rate, known AI crawlers. Regular users are not affected.

üé´

Token System

After solving PoW, a temporary token is issued. No need to solve challenge for every request ‚Äî only on first contact.

üìä

Analytics

Metrics: challenges issued, solve rate, average time. Bot detection by anomalously fast/slow solving.
