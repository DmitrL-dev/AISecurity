## Модуль 1.1: Введение в IMMUNE

### Что такое IMMUNE?

IMMUNE — это био-вдохновленная система безопасности, работающая на уровне ядра операционной системы. Подобно иммунной системе человека, IMMUNE:

- **Обнаруживает** угрозы (патогены)
- **Запоминает** известные угрозы (иммунологическая память)
- **Адаптируется** к новым угрозам (адаптивный иммунитет)
- **Делится** знаниями с сетью (стадный иммунитет)

### Архитектура

```
┌─────────────────────────────────────┐
│           IMMUNE Hive               │
│  (Центральный командный сервер)     │
├─────────────────────────────────────┤
│  • Сбор данных от агентов           │
│  • Корреляция угроз                 │
│  • Распределение сигнатур           │
│  • Управление реагированием         │
└───────────────┬─────────────────────┘
                │
    ┌───────────┼───────────┐
    │           │           │
┌───▼───┐   ┌───▼───┐   ┌───▼───┐
│ Agent │   │ Agent │   │ Agent │
│ (ASM) │   │ (ASM) │   │ (ASM) │

```

## Модуль 1.2: Установка и настройка

### Требования

- DragonFlyBSD 6.0+
- 512 MB RAM минимум
- 100 MB дисковое пространство
- Сетевое подключение к Hive

### Установка агента

```bash
# Скачать и распаковать
fetch https://immune.sentinel.security/releases/agent-0.1.tar.gz
tar xf agent-0.1.tar.gz
cd immune-agent

# Установить
sudo ./install.sh

# Проверить статус
service immuned status
```

### Конфигурация

Файл: `/etc/immune/agent.conf`

```conf
# Адрес Hive сервера
hive_address 192.168.1.100
hive_port 9998

# Уровень логирования (0-3)
verbose 1
```

## Модуль 1.3: Мониторинг

### Web Dashboard

Открыть в браузере: `http://hive-server:9999/dashboard`

### Ключевые метрики

| Метрика | Описание |
|---------|----------|
| Agents Online | Количество активных агентов |
| Threats (24h) | Угрозы за последние 24 часа |
| Herd Signatures | Сигнатуры стадного иммунитета |
| Isolated Hosts | Изолированные хосты |

### CLI команды

```bash
# Статус системы
immune status

# Список агентов
immune agents list

# Недавние угрозы
immune threats --last 24h
```

## Модуль 1.4: Реагирование на инциденты

### Уровни угроз

| Уровень | Название | Действие |
|---------|----------|----------|
| 1 | LOW | Только логирование |
| 2 | MEDIUM | Алерт + логирование |
| 3 | HIGH | Автоматическое блокирование |
| 4 | CRITICAL | Изоляция хоста |

### Действия оператора

1. **Просмотр алертов** → Dashboard или CLI
2. **Анализ инцидента** → Просмотр timeline
3. **Подтверждение** → Acknowledge alert
4. **Эскалация** → При необходимости
